# OpenCode 双 AI 自动化工作流配置文档

## 一、你的原有手动流程

1. Kimi 2.5 完成项目代码编写
2. Copilot GPT5.4 审核代码，输出改进建议文档
3. Kimi 读取审核文档内容
4. Kimi 根据建议自检、修补、优化代码

## 二、OpenCode 全自动闭环实现方案

全程无需切换编辑器、无需手动复制粘贴、无需额外脚本，OpenCode 原生工作流一键自动化执行。

### 1. 提前模型配置

在 OpenCode 设置 → AI 模型管理，添加绑定两个模型

- 模型 A：Kimi 2.5
- 模型 B：Copilot GPT-5.4

## 三、四节点自动化工作流（完整配置）

### 节点 1：Kimi 2.5 生成项目代码

- 选用模型：Kimi 2.5
- 触发方式：手动启动 / 文件保存自动触发
- 提示词模板：
    
    完成当前项目的代码编写，遵循项目需求，输出完整可运行代码。
    
    文件路径：{{当前文件}}
- 输出行为：自动写入对应代码文件

---

### 节点 2：Copilot GPT5.4 代码评审 + 生成优化文档

- 选用模型：Copilot GPT-5.4
- 执行顺序：节点 1 执行完成后**自动运行**
- 提示词模板：
    
    你是专业代码审核工程师。
    
    严格审核下方全部项目代码：
    
    {{读取当前文件内容}}

审核输出规范：

1. 潜在 Bug、逻辑问题
2. 性能短板、内存与效率问题
3. 代码规范、可读性问题
4. 安全风险隐患
5. 可直接落地替换的优化代码
    
    最终整理为标准 Markdown 格式审核报告

- 输出行为：自动生成 `code_review.md` 审核文档

---

### 节点 3：Kimi 2.5 读取并加载评审建议

- 选用模型：Kimi 2.5
- 执行顺序：节点 2 完成后自动执行
- 提示词模板：
    
    完整读取并理解这份代码审核优化建议：
    
    {{读取 code_review.md 全部内容}}
    
    请确认已全部理解，无需额外输出内容，仅载入上下文记忆。
- 输出行为：仅上下文加载，不写入文件

---

### 节点 4：Kimi 2.5 依据建议自动修复优化代码

- 选用模型：Kimi 2.5
- 执行顺序：节点 3 完成后自动执行
- 提示词模板：
    
    严格按照上文代码审核建议，对现有源码进行全量整改：

1. 修复所有已知 Bug
2. 优化运行性能
3. 统一代码格式规范
4. 补充注释、加固逻辑安全

原代码：

{{读取当前文件完整内容}}

直接输出修复完毕的完整版代码，覆盖原文件。

- 输出行为：自动覆盖更新原代码文件

## 四、一键运行效果

1. 点击运行工作流
2. 全自动走完：写代码 → AI 评审 → 生成报告 → 自动改代码
3. 全程无需人工干预、无需切换模型、无需复制文档

## 五、无感全自动进阶设置

工作流触发器 → 选择【文件保存时自动运行】

按下 Ctrl+S 保存代码，自动触发整套双 AI 审核优化闭环。

## 六、可直接导入 JSON 工作流配置

json

```
{
  "name": "Kimi+Copilot双AI自动化闭环",
  "description": "Kimi写代码 → Copilot审核 → Kimi自动修复",
  "trigger": "manual",
  "nodes": [
    {
      "model": "kimi-2.5",
      "prompt": "完成当前项目代码编写，文件路径：{{currentFile}}",
      "output": "write_to_file"
    },
    {
      "model": "copilot-gpt5.4",
      "prompt": "审核代码：{{readFileContent}}\n输出审核报告到code_review.md",
      "output": "write_to_file:code_review.md"
    },
    {
      "model": "kimi-2.5",
      "prompt": "读取并理解建议：{{readFile:code_review.md}}",
      "output": "context_only"
    },
    {
      "model": "kimi-2.5",
      "prompt": "根据建议修复代码：{{readFileContent}}\n直接输出完整修复代码",
      "output": "write_to_file:override"
    }
  ]
}
```

## 七、方案优势

1. 全程在 OpenCode 内完成，不跳转 VSCode 与网页
2. 双 AI 自动上下文联动，无需手动传递内容
3. 自动生成评审文档、自动覆盖修改源码
4. 通用所有项目，一次配置永久复用
5. 无额外插件、无自建脚本，轻量化稳定
